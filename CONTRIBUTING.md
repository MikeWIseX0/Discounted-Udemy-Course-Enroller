# Contributing to DUCE

We welcome and encourage community contributions, including issues, bug reports, feature requests, and pull requests! Below are the guidelines for contributing to the Discounted Udemy Course Enroller project.

---

## Getting Started

- **Discussions and Support**: Join our Discord Server to connect with the community or get technical help.
- **Reporting Issues**: Check the GitHub Issues tab to ensure your bug or feature request has not already been reported.

---

## Bug Reports

Please provide as much context as possible to help us diagnose and fix issues:
1. **Application Version**: State whether you are using the GUI or CLI version and the release version (e.g., v3.0.0).
2. **Operating System**: Specify your OS and version.
3. **Log Output**: Include log details from `duce_debug.log`.
4. **Steps to Reproduce**: Provide clear step-by-step instructions.

If you find a security-sensitive vulnerability, please report it privately to the maintainers instead of opening a public issue.

---

## Feature Requests

Suggestions for new features are always welcome. When proposing a feature, please describe:
- **The Problem**: What limitation or friction does this feature solve?
- **The Proposal**: How should the feature behave?
- **The Value**: Why is this feature beneficial for the project and its users?

---

## Pull Requests and Code Contributions

We welcome code contributions via pull requests! To ensure a smooth review process:
1. **Discuss First**: For major changes or new features, please open an issue first to align on the design.
2. **Coding Standards**: Adhere to PEP 8 standards. Format your changes using formatting tools such as `autopep8` or `black`, and verify code quality with `flake8`.
3. **Testing**: Run the unit test suite (`python -m unittest discover -s tests`) to verify that all tests pass before submitting.
4. **Clean Commits**: Write descriptive, professional commit messages detailing the changes.

Thank you for contributing to DUCE!
